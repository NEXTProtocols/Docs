## Endpoint

```
POST /c4rust/alt
```

## Description

Generates SEO-optimized title and alt text for images using a vision-capable LLM via OpenRouter. Supports multiple detail levels and languages for flexible output.

## Authentication

This endpoint does not require authentication.

## Request Body

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `image_base64` | `string` | Yes | - | Base64 encoded image |
| `want_title` | `boolean` | No | `true` | Generate title |
| `want_alt` | `boolean` | No | `true` | Generate alt text |
| `alt_level` | `string` | No | `"standard"` | Detail level |
| `language` | `string` | No | `"en"` | Output language |
| `model_openrouter` | `string` | No | `"x-ai/grok-4.1-fast"` | Vision model |

### Alt Levels

| Level | Title Length | Alt Length | Use Case |
|-------|--------------|------------|----------|
| `short` | 20-40 chars | 40-80 chars | Thumbnails, icons |
| `standard` | 40-60 chars | 80-125 chars | Most images |
| `detailed` | 50-80 chars | 125-250 chars | Complex images, products |

## Example Request

<CodeGroup>
```bash curl
curl -X POST http://localhost:8000/c4rust/alt \
  -H "Content-Type: application/json" \
  -d '{
    "image_base64": "/9j/4AAQSkZJRgABAQAAAQABAAD...",
    "want_title": true,
    "want_alt": true,
    "alt_level": "standard",
    "language": "en",
    "model_openrouter": "x-ai/grok-4.1-fast"
  }'
```

```python python
import requests
import base64

# Read and encode image
with open("product.jpg", "rb") as f:
    image_base64 = base64.b64encode(f.read()).decode()

response = requests.post(
    "http://localhost:8000/c4rust/alt",
    json={
        "image_base64": image_base64,
        "want_title": True,
        "want_alt": True,
        "alt_level": "detailed",
        "language": "en"
    }
)
print(response.json())
```
</CodeGroup>

## Response

### Success Response (200)

```json
{
  "title": "Professional Laptop on Modern Desk Setup",
  "alt": "A sleek silver laptop open on a minimalist wooden desk, surrounded by a wireless mouse, coffee cup, and potted succulent plant, with natural light streaming through nearby windows"
}
```

### Title Only Response

```json
{
  "title": "Professional Laptop on Modern Desk Setup",
  "alt": null
}
```

### Alt Only Response

```json
{
  "title": null,
  "alt": "A sleek silver laptop open on a minimalist wooden desk..."
}
```

### Neither Requested

```json
{
  "title": null,
  "alt": null
}
```

### Error Response (400)

```json
{
  "detail": "Error processing image. Make sure the image format is supported (JPEG, PNG, GIF, WebP). Error: ..."
}
```

## Supported Image Formats

| Format | Magic Bytes |
|--------|-------------|
| JPEG | `FF D8 FF` |
| PNG | `89 50 4E 47` |
| GIF | `GIF87a` / `GIF89a` |
| WebP | `RIFF....WEBP` |

## Alt Level Guidelines

### Short Level
- Keep it very concise and direct
- Focus only on the main subject
- Use simple, clear words
- No unnecessary details

### Standard Level
- Be descriptive but concise
- Include the main subject and key context
- Use natural, readable language
- Balance between brevity and informativeness

### Detailed Level
- Provide rich, comprehensive descriptions
- Include main subject, context, colors, mood
- Describe visual elements that enhance understanding
- Use vivid but natural language

## SEO Best Practices

The generated text follows SEO best practices:

1. **Natural Language** - No keyword stuffing
2. **Descriptive** - Clearly describes visual content
3. **Appropriate Length** - Respects character limits
4. **Accessibility** - Useful for screen readers
5. **Relevant** - Focuses on what's important in the image

## Vision Models

| Model | Description |
|-------|-------------|
| `x-ai/grok-4.1-fast` | Default, fast and capable |
| `openai/gpt-4-vision-preview` | High quality |
| `anthropic/claude-3-sonnet` | Balanced quality |
| `google/gemini-pro-vision` | Google's vision model |

## Notes

- Images can be provided with or without data URL prefix
- Format is auto-detected from image magic bytes
- Both title and alt are optional - request only what you need
- The endpoint cleans markdown code blocks from LLM responses
